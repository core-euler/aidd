# DNA Map проекта

## Назначение

DNA Map связывает требования, документацию этапов и тесты.
Для каждого требования должно быть понятно:

- Где оно описано
- Где реализовано
- Чем проверяется

Если требование нельзя связать с тестом — оно не защищено.

---

## 2. Проверка полноты

Для каждого требования:

- Есть ли описание в spec.md?
- Есть ли описание логики в соответствующем этапе?
- Есть ли тест, подтверждающий выполнение?

Если хотя бы одного пункта нет — требование считается неполным.

---

## 3. Проверка устойчивости к изменениям

При изменении требования:

1. Найти строку в таблице
2. Проверить связанные этапы
3. Проверить связанные тесты
4. Убедиться, что изменение локализовано

Если изменение расползается на несвязанные модули —
декомпозиция требует пересмотра.
