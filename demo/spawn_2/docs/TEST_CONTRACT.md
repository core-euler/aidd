# Test Contract

## Цель
Проверить, что реализованный бот полностью соответствует SPEC.md и связанным документам. Тесты формулируются в терминах поведения и ожидаемых результатов, без привязки к коду.

## Общие правила
- Проверка выполняется в таймзоне, указанной в окружении (TIMEZONE).
- Дедлайны сохраняются в UTC и отображаются в локальной таймзоне.
- Все действия выполняются через inline‑кнопки, кроме /start.

## Базовые тесты запуска
1. /start создаёт персонажа со стартовыми значениями и показывает кнопку создания первой задачи.
2. Повторный /start открывает главное меню.
3. Все кнопки меню открывают соответствующие экраны.

## Создание задачи
1. Пустое название отклоняется с повторным запросом.
2. Название > 200 символов обрезается и пользователь уведомляется.
3. Выбор сложности обязателен, после выбора открывается выбор дедлайна.
4. Быстрые дедлайны рассчитываются корректно относительно локального времени.
5. Ручной ввод принимает все форматы из SPEC.md.
6. Дедлайн в прошлом отклоняется.
7. После создания задачи выводится подтверждение и кнопки действий.

## Список задач
1. Список отображается сортированным по дедлайну.
2. Каждая задача открывает детальный экран.
3. При отсутствии задач выводится информативное сообщение.
4. Кнопка «Просроченные» показывает список просроченных задач.

## Детали задачи
1. Отображаются: название, сложность, дедлайн, оставшееся время.
2. «Выполнено» завершает задачу и начисляет опыт при условии дедлайна в будущем.
3. «Удалить» удаляет задачу без опыта и урона.

## Игровая механика
1. Начисление XP соответствует сложности.
2. При достижении порога происходит повышение уровня.
3. Излишек опыта переносится.
4. Множественные повышения уровней за одну задачу корректны.

## Просрочка
1. Проверка дедлайнов каждые 5 минут.
2. Просрочка переводит задачу в статус failed и наносит урон.
3. Урон наносится только один раз на задачу.
4. При нескольких просрочках урон суммируется.

## Смерть персонажа
1. При HP <= 0 прогресс сбрасывается.
2. Все активные задачи удаляются.
3. Пользователь получает уведомление о смерти.

## Уведомления
1. Напоминание за 1 час отправляется один раз.
2. Уведомления о просрочке, смерти и повышении уровня отправляются корректно.

## Таймзона
1. Созданная «через 1 час» задача имеет дедлайн в UTC+1 час от локального времени.
2. В интерфейсе дедлайн отображается в локальном времени.
3. Просрочка определяется корректно относительно UTC‑хранения.
