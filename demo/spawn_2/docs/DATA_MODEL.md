# Модель данных

## Пользователь
Сущность отражает персонажа и статистику пользователя.

Поля и ограничения:
- Идентификатор пользователя в Telegram: уникален и обязателен
- Имя пользователя Telegram: может отсутствовать
- Уровень: целое число, стартовое значение 1
- Опыт: целое число, стартовое значение 0
- Текущее здоровье: целое число, стартовое значение 100
- Максимальное здоровье: целое число, стартовое значение 100
- Всего выполнено задач: целое число, стартовое значение 0
- Всего просрочено задач: целое число, стартовое значение 0
- Максимальный достигнутый уровень: целое число, стартовое значение 1
- Дата регистрации

Инварианты:
- Текущее здоровье не превышает максимальное
- Уровень не меньше 1
- Опыт не отрицательный

## Задача
Сущность отражает задачу пользователя.

Поля и ограничения:
- Связь с пользователем: обязательна
- Название: обязательное, длина до 200 символов
- Сложность: одно из четырёх значений
- Дедлайн: обязателен, не в прошлом при создании
- Дедлайн хранится в UTC
- Статус: активная, выполненная, просроченная, удалённая
- Дата создания
- Дата выполнения: заполняется только при выполнении
- Флаг напоминания: отражает отправку уведомления за 1 час

## Справочник сложностей

| Сложность | Опыт за выполнение | Урон за просрочку |
|---|---|---|
| лёгкая | 10 | 5 |
| средняя | 25 | 15 |
| сложная | 50 | 30 |
| эпическая | 100 | 50 |

## Статусы задач

| Статус | Значение | Описание |
|---|---|---|
| активная | active | задача ожидает выполнения |
| выполненная | completed | задача выполнена вовремя |
| просроченная | failed | дедлайн прошёл, урон нанесён |
| удалённая | deleted | пользователь удалил задачу |
