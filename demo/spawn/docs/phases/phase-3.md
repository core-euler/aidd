# Этап 3: Игровая механика

## Цель
Добавить начисление опыта, систему уровней, проверку дедлайнов и урон за просрочку.

## Входные условия
- Реализовано управление задачами.
- Данные задач и пользователей корректно сохраняются.

## Требования
- R3: выполнение задачи и начисление опыта при соблюдении дедлайна.
- R6: корректные показатели уровня, опыта и здоровья.
- NR2: проверка дедлайнов каждые 5 минут.
- E5: выполнение просроченной задачи без опыта.
- E8–E13: урон, смерть, уровни и перенос опыта.
- E9: суммарный урон при одновременной просрочке.
- E10: корректность при конкурентных действиях.

## Компоненты
- Сервисный слой задач.
- Сервисный слой персонажа.
- Модуль планировщика.
- Слой доступа к данным.

## Логика работы
- При выполнении активной задачи начисляется опыт согласно сложности.
- Опыт повышает уровень по формуле из спецификации, излишек переносится.
- Максимальное здоровье увеличивается при повышении уровня, текущее здоровье восстанавливается.
- Планировщик каждые 5 минут проверяет активные задачи.
- При просрочке задача переводится в статус просроченной, наносится урон, статистика обновляется.
- Урон по одной задаче применяется только один раз.
- При одновременной просрочке нескольких задач урон суммируется.

## Критерии готовности
- Начисление опыта и повышение уровней работают на всех сложностях.
- Перенос излишка опыта и множественные уровни за одну задачу корректны.
- Просрочка задач наносит урон и меняет статус.
- Планировщик обрабатывает дедлайны с периодичностью 5 минут.
