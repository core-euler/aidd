# Этап 1: Базовый бот

## Цель
Запустить базовую версию бота: первый запуск, главное меню и экран персонажа.

## Входные условия
- Настроены переменные окружения для токена бота и подключения к базе данных.
- База данных доступна и готова принимать данные пользователя.
- В проекте есть каркас бота с обработкой входящих событий.

## Требования
- R12: единственная команда первого запуска, остальная навигация через кнопки.
- R13: главное меню и базовые кнопки.
- R17: экран персонажа.
- R19: кнопки навигации назад или в меню.
- E7: информативные сообщения при отсутствии данных на экранах.

## Компоненты
- Модуль входящих событий.
- Модуль сценариев и экранов.
- Сервисный слой персонажа.
- Слой доступа к данным.

## Логика работы
- Команда первого запуска создаёт пользователя со стартовыми параметрами.
- Повторный запуск открывает главное меню без создания новой записи.
- Главное меню отображает краткий статус персонажа и основные кнопки.
- Экран персонажа показывает уровень, опыт, здоровье, количество активных задач и ближайший дедлайн, если он есть.
- Все переходы выполняются через inline-кнопки и возвращают пользователя в меню или назад.

## Критерии готовности
- Первый запуск создаёт персонажа и показывает приветствие с кнопкой создания первой задачи.
- Повторный запуск открывает главное меню.
- Экран персонажа корректно отображает базовые характеристики.
- Навигация между меню и экраном персонажа работает через кнопки.
