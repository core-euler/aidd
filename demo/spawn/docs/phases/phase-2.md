# Этап 2: Управление задачами

## Цель
Реализовать создание задач, просмотр списка и экран деталей задачи с действиями.

## Входные условия
- Работает базовый бот и главное меню.
- Сущности пользователя и задачи доступны в базе данных.

## Требования
- R1: создание задачи с названием, сложностью и дедлайном.
- R2: список активных задач.
- R3: отметка задачи выполненной.
- R4: удаление задачи без опыта и урона.
- R5: список просроченных задач.
- R14: пошаговый процесс создания.
- R15: быстрые варианты дедлайна.
- R16: экран задачи с действиями.
- R19: кнопки навигации.
- R20: форматы ручного ввода дедлайна.
- E1–E7: edge cases создания и действий с задачами.
- E14–E15: множественные задачи и большой объём.

## Компоненты
- Модуль входящих событий.
- Модуль сценариев и экранов.
- Сервисный слой задач.
- Слой доступа к данным.

## Логика работы
- Создание задачи выполняется шагами: название, сложность, дедлайн.
- Название валидируется на пустое значение и длину, при превышении длины обрезается с уведомлением.
- Дедлайн в прошлом отклоняется, при ошибке формата запрашивается повторный ввод с подсказками.
- Быстрые дедлайны рассчитываются относительно локального времени пользователя.
- Список задач отображается отсортированным по дедлайну.
- Детальный экран задачи показывает название, сложность, дедлайн и оставшееся время.
- Выполнение активной задачи завершает её без начисления опыта на этом этапе.
- Удаление задачи меняет статус на удалённую без начисления опыта и урона.
- Отдельная кнопка показывает просроченные задачи.

## Критерии готовности
- Можно создать задачу по всем вариантам дедлайна.
- Список активных задач отображается кнопками и открывает детали.
- Детальный экран позволяет выполнить или удалить задачу.
- Просроченные задачи отображаются отдельным списком.
- Граничные случаи из требований корректно обработаны.
