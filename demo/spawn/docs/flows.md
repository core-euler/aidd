# Сценарии и состояния

## Основные сценарии

### Первый запуск
- Пользователь отправляет команду запуска
- Бот создаёт персонажа со стартовыми значениями
- Бот показывает приветствие и предлагает создать первую задачу

### Создание задачи
- Пользователь выбирает создание новой задачи
- Бот запрашивает название
- После ввода названия бот предлагает выбрать сложность
- После выбора сложности бот предлагает выбрать дедлайн
- Пользователь выбирает быстрый дедлайн или вводит вручную
- Бот подтверждает создание задачи и возвращает в меню

### Просмотр активных задач
- Пользователь открывает список задач
- Бот показывает задачи кнопками и кнопку просмотра просроченных
- Пользователь выбирает задачу и видит детали
- Пользователь может отметить задачу выполненной или удалить

### Просмотр персонажа
- Пользователь открывает экран персонажа
- Бот показывает уровень, опыт, здоровье и ближайший дедлайн

### Просмотр статистики
- Пользователь открывает экран статистики
- Бот показывает выполненные, просроченные задачи, успешность, максимальный уровень

### Просрочка задачи
- Планировщик обнаруживает просрочку
- Бот наносит урон и уведомляет пользователя
- При падении здоровья до нуля прогресс сбрасывается и активные задачи удаляются

## Состояния задачи

Состояния:
- активная
- выполненная
- просроченная
- удалённая

Переходы:
- активная становится выполненной при отметке пользователем до дедлайна
- активная становится просроченной при наступлении дедлайна
- активная становится удалённой при удалении пользователем
- выполненная, просроченная и удалённая не возвращаются в активную

## Состояния пользователя

Состояния:
- живой персонаж
- смерть и сброс прогресса

Переходы:
- живой персонаж получает опыт за выполнение задач
- живой персонаж получает урон при просрочке
- при здоровье ноль или ниже происходит сброс прогресса

## Обработка граничных случаев
- Дедлайн в прошлом при создании — отказ с сообщением
- Пустое название — отказ с сообщением
- Название длиннее 200 символов — обрезка и уведомление
- Некорректный формат даты — повторный запрос с примерами
- Выполнение просроченной задачи — опыт не начисляется
- Несколько просроченных задач в один момент — урон суммируется
