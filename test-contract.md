# Test Contract

## Назначение

Этот документ описывает, какие сценарии должны быть протестированы.
Это не реализация тестов, а перечень проверяемого поведения.

---

## 1. Functional Scenarios

### F1 — Успешная авторизация
Вход:
- валидные данные пользователя

Ожидаемый результат:
- пользователь получает доступ
- создаётся сессия

---

### F2 — Ошибка авторизации
Вход:
- неверный пароль

Ожидаемый результат:
- отказ в доступе
- информативное сообщение

---

## 2. Edge Cases

- Пустые значения
- Повторная регистрация
- Одновременные запросы

---

## 3. Негативные сценарии

- Попытка доступа без авторизации
- Неверный формат данных

---

## 4. Нефункциональные проверки

- Время ответа не превышает X
- Ошибки логируются
